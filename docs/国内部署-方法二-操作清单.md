# 国内部署 方法二：完整操作清单（自动模式用真实长江有色价格）

按顺序做下面几步，做完即可通过国内网址打开，且自动模式会拉取真实铜/银价格。

---

## 准备：先在本机打包出 dist

在项目根目录（有 `package.json` 的文件夹）打开终端，执行：

```bash
npm install
npm run build
```

执行完后会多出一个 **`dist`** 文件夹，里面就是网站要上传的文件。后面步骤会用到。

---

## 第一步：腾讯云 COS 静态网站（放前端）

1. 打开浏览器，登录 [腾讯云控制台](https://console.cloud.tencent.com/cos) → 左侧 **对象存储** → **存储桶列表** → **创建存储桶**。
2. **名称**：自己起一个（如 `bauchi-profit`），**地域**选国内（如「广州」），**访问权限**选「公有读私有写」→ 点击**确定**。
3. 创建好后点进这个桶 → 左侧 **基础配置** → 找到 **静态网站** → 点**编辑**，**开启**静态网站，**索引文档**填 `index.html`，**错误文档**也填 `index.html` → **保存**。
4. 左侧点 **文件列表** → **上传文件** → 选**本地的 `dist` 文件夹里的所有内容**（即 `index.html` 和 `assets` 文件夹一起选上），上传到当前桶的**根目录**（不要选整个 `dist` 文件夹上传，否则会多一层路径）。
5. **重要**：2024 年后默认域名在浏览器里会变成下载而非打开网页，需要绑定**自定义域名**才能正常访问。详细步骤见：[COS自定义域名-让网站在浏览器里打开.md](COS自定义域名-让网站在浏览器里打开.md)。

---

## 第二步：腾讯云 SCF 云函数（价格接口）

1. **打包云函数代码**  
   打成 zip，且 **zip 打开后根目录就是 `index.js`**（不要是 `ccmn-prices/index.js`）。  
   - **Windows**：打开文件夹 `scf\ccmn-prices`，选中 `index.js` → 右键「发送到」→「压缩(zipped)文件夹」，得到 `index.zip`，可改名为 `ccmn-prices.zip`。  
   - **Mac/Linux**：在项目根目录执行  
     ```bash
     cd scf/ccmn-prices && zip -r ../../ccmn-prices.zip index.js && cd ../..
     ```
     会在项目根目录得到 `ccmn-prices.zip`，zip 内根目录即 `index.js`。

2. **创建云函数**  
   - 登录 [腾讯云 SCF 控制台](https://console.cloud.tencent.com/scf) → **函数服务** → **新建**。
   - **函数名称**：`ccmn-prices`（或任意）。
   - **运行环境**：**Node.js 16.13** 或 **Node.js 18.15**。
   - **提交方法**：本地上传 zip，选刚打好的 `ccmn-prices.zip`。
   - **入口**：填 `index.main_handler`（不要改）。
   - 创建。

3. **创建函数 URL**（推荐，API 网关已逐步停服）  
   - 进入该函数 → 左侧 **函数 URL**（或「触发管理」里找「函数URL」）→ **新建函数 URL**。
   - **别名/版本**：默认流量即可。
   - **公网/内网**：选 **公网**，方便网页访问。
   - **授权类型**：选 **开放**（匿名访问）。
   - **CORS**：可开启并允许 `*`，方便前端跨域。
   - 提交后会出现一个 **URL 地址**，形如：`https://xxxxx.gz.tencentscf.com`。
   - **复制这个完整地址**，下一步在 `.env.production` 里填 **`VITE_CCMN_API_URL=该地址`**（见第三步）。

   **若你仍使用 API 网关**（不推荐）：触发方式选「API 网关触发」，路径填 `/api/ccmn-prices`，拿到网关域名后填 **`VITE_CCMN_API_BASE=域名`**（不要带路径）。

---

## 第三步：前端填 API 地址并重新打包上传

1. **配置环境变量**  
   在项目**根目录**（和 `package.json` 同级）：
   - 复制 `.env.production.example` 为 **`.env.production`**（没有 example 就新建）。
   - **用函数 URL 时**：在 `.env.production` 里填（把地址换成你复制的函数 URL）：
     ```bash
     VITE_CCMN_API_URL=https://xxxxx.gz.tencentscf.com
     ```
   - **用 API 网关时**：填（只填域名，不要带路径）：
     ```bash
     VITE_CCMN_API_BASE=https://xxx-xxx.apigw.tencentcs.com
     ```
   - 保存。

2. **重新打包**  
   在项目根目录执行：
   ```bash
   npm run build
   ```

3. **覆盖上传到 COS**  
   把新生成的 **`dist`** 里的**全部文件**再上传到第一步的 COS 桶**根目录**，选择「覆盖」已有文件。

---

## 第四步：验证

1. 用浏览器打开你的 **COS 访问域名**（第一步记下的 `https://xxx.cos.xxx.myqcloud.com`）。
2. 切到 **自动模式**，看铜价/银价是否在几秒内变为真实数据（非“模拟数据”提示）。
3. 若一直显示模拟或报错，可：
   - 在浏览器 F12 → Network，看是否有请求 `你的API网关地址/api/ccmn-prices`，以及返回状态码和内容；
   - 在 SCF 控制台该函数的「日志」里看是否有报错。

---

## 之后更新网站

- 只改前端：改代码 → `npm run build` → 把新 `dist` 覆盖上传到 COS。
- 只改云函数：改 `scf/ccmn-prices/index.js` → 重新打 zip → 在 SCF 控制台该函数「代码」里上传新 zip。
- API 网关地址没变则不用改 `.env.production`。
